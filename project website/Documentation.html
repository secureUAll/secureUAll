Actors

Admin

The admin has access to the entire system and its features.

Owner

The owner can manage her/his machines, check statistics/vulnerabilities found, define a scrapping level for a specific machine, and receives notifications whenever a vulnerability is found.

User

The user can check global statistics about the system, and can request to receive notifications about a specific machine.

Requirements

Functional

  • Dashboard
  • Web platform available 24/7 through UA IDP access that allows the system management and vulnerability monitoring.
  • Scrapping
  • With a configurable periodicity and with several levels (from the less to the most evasive), it recurs to external pluggable tools.
  • Alerts
  • Email sent every time a vunerability is detected. It informs the manager that of its existance and suggests possible corrections.
  • Vulnerability Analysis
  • Based on the data gathered by scrapping tools, the risk level is computed based on CVSS and the vulnerability scan.

Non Functional

  • Scalability
  • The work is distributed by workers in order to respond to high demand.
  • Portability
  • A container implementation allows a simple deploy in different execution environments.
  • Usability
  • An intuitive interface for management and control that allows a good user experience.
  • Privacy
  • Ensure that personal data eventually obtained during scans is not stored.
  • Security
  • Only authorized people (administrators, owners and subscribers) have access to machine related data and reports.
